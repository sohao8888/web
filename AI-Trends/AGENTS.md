# AI趋势知识库运行规则

## 一、目标

这个知识库用于长期追踪 AI 行业趋势，服务于研究、判断和内容创作。

系统的核心任务有三件事：

1. 收集高价值信息
2. 提炼结构化趋势信号
3. 生成可复用的日报、周报、月报和主题判断

所有输出默认使用中文。

---

## 二、目录规则

### 00_Inbox
存放原始剪藏、临时记录、未整理材料。

### 01_Sources
存放清洗后的来源笔记。
每篇 source note 必须保留原始链接、来源名称、日期、摘要。

### 02_Signals
存放从 source note 中提炼出的趋势信号。
每条 signal 必须说明变化点、重要性、影响对象、后续观察点。

### 03_Themes
存放长期主题页，例如：
AI代理
推理模型
多模态
语音
视频
AI搜索
基础设施
商业化

### 04_Companies
存放公司追踪页，例如：
OpenAI
Anthropic
Google
Meta
Microsoft
NVIDIA
Amazon

### 05_Weekly
存放每周趋势周报。

### 06_Monthly
存放每月趋势月报。

### 07_Daily
存放每日日报。

### 90_Templates
存放模板文件。

### 99_Dashboard
存放总览页面和知识库入口页。

---

## 三、写作规则

1. 全部使用中文
2. 用词清楚直接
3. 摘要优先信息密度
4. 区分事实、解释、预测
5. 不使用夸张营销腔
6. 不把原始来源和个人判断混在一起
7. 不删除原始链接
8. 不凭空补造来源中没有的信息

---

## 四、元数据规则

所有笔记尽量使用统一的 YAML frontmatter。

至少应包含以下字段：

- type
- title
- date
- status
- company
- theme
- signal_type
- importance
- confidence
- summary
- my_take
- related_notes
- tags

### 字段说明

#### type
可选值：
source
signal
theme
company
daily
weekly
monthly
dashboard

#### status
可选值：
inbox
draft
reviewed
archived

#### signal_type
可选值：
capability
product
business
infra
policy

#### importance
取值范围：
1 到 5

#### confidence
取值范围：
1 到 5

---

## 五、Source Note 规则

每篇 source note 必须回答以下问题：

1. 这条信息来自哪里
2. 这条信息讲了什么
3. 它属于哪个公司或主题
4. 它可能对应哪种趋势信号

每篇 source note 需要：
- 保留 source_url
- 保留 source_name
- 提取发布日期
- 写三句话中文摘要
- 尽量标注 company 和 theme

---

## 六、Signal Note 规则

每条 signal note 必须回答以下问题：

1. 到底发生了什么变化
2. 这属于哪类信号
3. 为什么重要
4. 影响谁
5. 接下来该观察什么

Signal note 不要只是重复原文摘要，必须有提炼和归纳。

每条 signal 应尽量链接回对应的 source notes。

---

## 七、Theme Page 规则

主题页用于长期跟踪某个方向，不用于记录单次新闻。

每个主题页至少包含：

1. 主题定义
2. 为什么重要
3. 相关公司
4. 近期信号
5. 长期判断
6. 待观察问题

主题页要尽量链接相关 source 和 signal。

---

## 八、Company Page 规则

公司页用于长期跟踪单个公司。

每个公司页至少包含：

1. 公司简介
2. 核心方向
3. 最近重要动作
4. 涉及的主题
5. 我的阶段性判断
6. 相关 source 和 signal

---

## 九、日报规则

日报用于记录当天最重要的变化。

每篇日报应包括：

1. 今日最重要的 3 条变化
2. 按公司整理今日动态
3. 按主题整理今日信号
4. 最值得继续跟踪的 5 个信号
5. 当天形成的初步判断
6. 可延展成内容的选题
7. 相关笔记链接

日报重在快和准，不追求面面俱到。

---

## 十、周报和月报规则

### 周报
周报用于总结最近 7 天的主要变化。
重点看趋势是否在连续出现。

### 月报
月报用于总结最近 30 天的主要方向。
重点看哪些变化正在形成更长期的结构性趋势。

周报和月报都应明确：
- 发生了什么
- 为什么重要
- 哪些方向值得继续押注
- 哪些热闹正在降温

---

## 十一、命名规则

文件名尽量清晰统一。

推荐格式：

### source
YYYY-MM-DD 来源名称 标题.md

### signal
YYYY-MM-DD 信号 关键词.md

### daily
YYYY-MM-DD AI趋势日报.md

### weekly
YYYY-Wxx AI趋势周报.md

### monthly
YYYY-MM AI趋势月报.md

### company
公司名.md

### theme
主题名.md

---

## 十二、编辑规则

1. 优先补全已有笔记，不要制造重复文件
2. 修改前先理解上下文
3. 不要批量重写用户的个人判断
4. 不要删除已有链接
5. 不要改坏 YAML 结构
6. 尽量保持格式统一
7. 新建笔记后要补 related_notes

---

## 十三、自动化任务规则

允许执行以下自动化任务：

1. 扫描 Inbox 并整理 source notes
2. 从 source notes 提炼 signal notes
3. 更新 theme 和 company 页面
4. 生成日报、周报、月报
5. 修复链接和补全元数据
6. 生成 dashboard 汇总页

不允许执行以下操作：

1. 未经明确要求删除笔记
2. 覆盖用户的主观判断
3. 编造来源
4. 将无来源内容写成事实
5. 擅自修改目录结构

---

## 十四、判断规则

在做趋势判断时，尽量从以下几个维度看：

1. 能力有没有实质提升
2. 产品有没有真正落地
3. 商业模式有没有变化
4. 基础设施有没有跟上
5. 这件事会影响谁
6. 这件事会持续多久

要尽量区分：
短期热闹
中期趋势
长期结构变化

---

## 十五、输出风格

输出默认遵循以下风格：

1. 简洁
2. 准确
3. 高信息密度
4. 少空话
5. 少套话
6. 面向长期研究和内容创作
7. 适合后续直接改写成公众号选题或内部判断稿

---

## 十六、安全规则

1. 不删除任何笔记，除非明确要求
2. 不覆盖原始来源链接
3. 不把猜测写成事实
4. 不对缺失信息进行无依据补全
5. 对信息不完整的笔记，status 设为 draft
6. 优先保留原始记录，再补充结构化内容

---

## 十七、执行优先级

执行任务时，优先级如下：

1. 保留原始资料
2. 保证结构化字段完整
3. 保证链接关系正确
4. 生成趋势信号
5. 生成日报、周报、月报
6. 优化主题页和公司页

所有操作都要以“可长期维护、可检索、可复盘”为目标。
