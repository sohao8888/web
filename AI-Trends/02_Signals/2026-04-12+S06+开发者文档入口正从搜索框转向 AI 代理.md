# 开发者文档入口正从搜索框转向 AI 代理

## 信号结论

开发者获取信息的主要入口正在从人类主动搜索转向代理主动读取，文档和 API 设计开始面向机器而不只是面向人。

## 1. 这条信息属于哪一类

这条信息属于：`产品`。

## 2. 为什么重要

如果代理已经占据大量文档流量，那么谁更容易被代理读取、理解和调用，谁就更有机会成为默认技术栈。

## 3. 影响谁

- 开发者工具公司，需要把文档、示例和 API 说明做成 agent-friendly。
- OpenAI、Anthropic 等模型平台，可能进一步成为新的分发层和默认入口。
- 文档与开发者关系团队，工作重心会从 SEO 扩展到结构化可读性和可调用性。

## 4. 未来 3 到 12 个月值得继续观察什么

- 代理来源流量占比是否在更多开发者网站上快速上升。
- 文档格式是否更明显地转向结构化、可检索和可调用。
- 面向代理的认证、限流、API 说明和 SDK 规范是否出现新标准。
